第一題 上機考

假設你正在開發一個ASP.NET Web Forms應用程序，其中使用Session物件來存儲和檢索用戶信息。現在，你需要創建一個簡單的登錄頁面，該頁面應該能夠註冊新用戶，並將他們的信息存儲在Session物件中。你需要完成以下任務：

1.創建一個名為"Login.aspx"的Web Forms頁面，該頁面包含兩個文本框（一個用於輸入用戶名，另一個用於輸入密碼）和一個按鈕（用於提交表單）。

2.在頁面的Page_Load事件中，檢查Session物件中是否存在已登錄用戶的信息。如果是，則重定向到另一個頁面（例如，"Home.aspx"），該頁面將顯示用戶的信息。

3.在按鈕的Click事件中，檢查用戶輸入的用戶名和密碼是否與預設值匹配。如果匹配，則將用戶的信息存儲在Session物件中，並重定向到"Home.aspx"頁面。否則，顯示一個錯誤消息。

4.創建一個名為"Home.aspx"的Web Forms頁面，該頁面顯示已登錄用戶的信息。在頁面的Page_Load事件中，檢查Session物件中是否存在用戶信息。如果是，則顯示用戶的信息。否則，重定向到"Login.aspx"頁面。

提示：你可以使用Session物件的屬性和方法來實現這些任務，例如Session["key"]屬性來設置和檢索Session物件的值，Session.Timeout屬性來設置Session物件的超時值，Session.Abandon方法來銷毀Session物件，等等

第四題 名詞
請說明關於 Application Session Cookies Cache ViewState ，內容須有變數儲存位置(伺服器 、客戶端)，生命週期(什麼時候不見)，實際例子
Ans-
Application: 
伺服器。
不關伺服器就一直存在。
特定資源的下載次數、共享計數器。

Session: 
伺服器。
可設定時間，直到超時或關閉網頁(預設20分鐘)。
使用者登入狀態。

Cookies: 
用戶端。
可設定時間，直到過期或被使用者刪除。

Cache: 
伺服器。
可設定時間，直到過期或被清除。
儲存靜態資源文件，如圖片、CSS和JavaScript文件。

ViewState: 
瀏覽器Web頁面HTML的隱藏欄位。
單一頁面的生命週期內。
同一頁面的不同請求之間恢復控制項狀態。




第五題 名詞
解釋什麼是Request物件？它有哪些常用屬性和方法？

Ans-
用於處理用戶端瀏覽器的HTTP Request。用戶對伺服器進行要求Request時，可使用此物件取得對伺服器所送出的資訊。

常用屬性:
QueryString：用於從 URL中擷取query string參數的值。
Cookies:獲取客戶端瀏覽器中的 Cookie。

常用方法: 
UserHostAddress:獲取客戶端的 IP 地址。





第六題 名詞
解釋什麼是Response物件？它有哪些常用屬性和方法？

Ans-
用於將資料從伺服器傳送回瀏覽器，包括網頁伺服器所要回應給使用者的各項資訊。

常用屬性:
Cookies：用於處理 HTTP cookie，設置和讀取瀏覽器的 cookie。
Status：設置或獲取 HTTP response 的status code。


常用方法: 
Redirect()：將用戶瀏覽器導向到另一個 URL。

Ans-
Request：Client對Server發出的請求

第八題 名詞
什麼是Page物件？它與控制項的生命週期有什麼關係？

Ans-
1.
一個  .aspx 檔對應到一個 Page 類別， 在Page 類別中，可定義控制項、處理事件等，以構建 Web 頁面的內容和行為。
Page 類別在網頁執行的時候，建立Page物件實體，Page 物件會繼承 .aspx 檔中定義的控制元素，處理事件，並生成 HTML 來呈現給用戶。


2.
1. Page_Init： 頁面被初始化。可以在這個階段建立控件。

2. Control_Init：控制項的初始化階段，用於初始化控制項，並在頁面的每個控件上執行。

3. Control_Load： 控制項的狀態和數據被載入。控制項的狀態是在這個階段從ViewState中還原的。

4. Page_Load：頁面的主要載入階段，用於處理用戶輸入、設定控件的數據和執行頁面初始化邏輯。通常用於讀取用戶輸入和設定控制項。

5. Button_Click：處理按鈕點擊事件的階段，用於執行與按鈕相關的業務邏輯。

6. Control_PreRender： 控制項的準備工作被完成，控制項的狀態被保存。


7. Page_PreRender：頁面的最後準備階段，用於修改頁面的內容、控制項的最後設定以及處理其他最後的工作。






第九題 名詞
什麼是Server物件？它可以用來做什麼？

Ans-
Server物件用於提供 ASP.NET取得伺服器提供的各項資訊。

常用屬性:
MachineName : 查詢伺服器名稱。
ScriptTimeout : 設定指令碼程式執行的時間上限(單位S(秒))。


常用方法: 
URLEncode() : 將網址字串轉換為URL編碼。
URLDecode() : 將URL編碼轉換為字串。
